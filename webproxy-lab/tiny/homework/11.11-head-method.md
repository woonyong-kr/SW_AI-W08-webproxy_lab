# 숙제 11.11 -- HTTP HEAD 메서드 지원

## 문제

Tiny를 확장하여 HTTP HEAD 메서드를 지원하라.
telnet을 웹 클라이언트로 사용하여 작동을 확인하라.

## 수정 내용

### 1. doit() 함수: HEAD 메서드 분기 추가

```c
int is_head = 0;

if (!strcasecmp(method, "HEAD")) {
    is_head = 1;
} else if (strcasecmp(method, "GET")) {
    clienterror(fd, method, "501", "Not implemented", ...);
    return;
}
```

### 2. serve_static() 함수: head_only 매개변수 추가

```c
void serve_static(int fd, char* filename, int filesize, int head_only) {
    // ... 헤더 전송 ...

    /* HEAD 요청이면 본문을 전송하지 않음 */
    if (head_only || filesize == 0) {
        return;
    }

    // ... 본문 전송 ...
}
```

### 3. 동적 콘텐츠의 HEAD 처리

HEAD 요청 시 CGI를 실행하지 않고 상태 라인과 서버 헤더만 반환한다.

## HEAD vs GET

| 항목 | GET | HEAD |
|------|-----|------|
| 응답 헤더 | 전송 | 전송 |
| 응답 본문 | 전송 | 전송하지 않음 |
| 용도 | 콘텐츠 조회 | 리소스 존재 확인, 메타데이터 조회 |

HEAD는 GET과 동일한 헤더를 반환하되 본문만 생략한다.
캐시 유효성 검사, 링크 검증, 리소스 크기 확인 등에 사용된다.

## 테스트 방법

```bash
$ ./tiny 8080

# 다른 터미널에서
$ telnet localhost 8080
HEAD /home.html HTTP/1.0
Host: localhost

```

예상 응답 (본문 없이 헤더만 반환):

```
HTTP/1.0 200 OK
Server: Tiny Web Server
Content-length: 120
Content-type: text/html
```

연결이 바로 종료되고, HTML 본문은 전송되지 않는다.
