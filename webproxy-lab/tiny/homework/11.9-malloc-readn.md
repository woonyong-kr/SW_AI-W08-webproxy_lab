# 숙제 11.9 -- mmap 대신 malloc + rio_readn 사용

## 문제

Tiny를 수정하여 정적 콘텐츠를 서빙할 때 mmap과 rio_writen 대신
malloc, rio_readn, rio_writen을 사용하도록 하라.

## 수정 내용

`serve_static()` 함수에서 파일을 메모리에 매핑하는 방식을 변경했다.

### 수정 전 (mmap 방식)

```c
srcfd = Open(filename, O_RDONLY, 0);
srcp = Mmap(0, (size_t)filesize, PROT_READ, MAP_PRIVATE, srcfd, 0);
Close(srcfd);
Rio_writen(fd, srcp, (size_t)filesize);
Munmap(srcp, (size_t)filesize);
```

### 수정 후 (malloc + rio_readn 방식)

```c
srcfd = Open(filename, O_RDONLY, 0);
srcp = (char *)Malloc((size_t)filesize);
Rio_readn(srcfd, srcp, (size_t)filesize);
Close(srcfd);
Rio_writen(fd, srcp, (size_t)filesize);
Free(srcp);
```

## 두 방식의 차이

| 항목 | mmap | malloc + read |
|------|------|---------------|
| 메모리 할당 | 가상 메모리 매핑 | 힙 메모리 할당 |
| 파일 읽기 | 페이지 폴트 시 자동 | 명시적 rio_readn 호출 |
| 메모리 해제 | Munmap | Free |
| 큰 파일 | 가상 주소 공간만 사용 | 전체를 힙에 복사 |
| filesize=0 처리 | mmap 실패 가능 | malloc(0)은 구현 정의 |
| 코드 복잡도 | 낮음 | 낮음 |

mmap은 커널이 페이지 캐시를 직접 매핑하므로 복사가 줄어든다.
malloc+read는 커널 -> 유저 공간 복사가 한 번 더 발생하지만 코드가 직관적이다.
