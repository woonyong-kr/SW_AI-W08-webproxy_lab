# 숙제 11.6c -- 브라우저 HTTP 버전 확인

## 문제

Tiny의 출력을 검사하여 브라우저가 사용하는 HTTP 버전을 확인하라.

## 방법

Tiny는 doit() 함수에서 요청 라인과 헤더를 printf로 출력한다.
서버를 실행하고 브라우저로 접속하면 터미널에 요청 내용이 출력된다.

```bash
$ ./tiny 8080
```

## 브라우저별 요청 예시

### telnet (수동 입력)

```
GET /home.html HTTP/1.0
Host: localhost
```

HTTP 버전: **HTTP/1.0** (사용자가 직접 입력한 대로)

### curl

```
$ curl http://localhost:8080/home.html
```

Tiny 출력:

```
Request headers:
GET /home.html HTTP/1.1
Host: localhost:8080
User-Agent: curl/7.81.0
Accept: */*
```

HTTP 버전: **HTTP/1.1**

### Chrome / Firefox 등 최신 브라우저

```
Request headers:
GET /home.html HTTP/1.1
Host: localhost:8080
Connection: keep-alive
Cache-Control: max-age=0
Upgrade-Insecure-Requests: 1
User-Agent: Mozilla/5.0 (X11; Linux x86_64) ...
Accept: text/html,application/xhtml+xml,...
Accept-Encoding: gzip, deflate, br
Accept-Language: ko-KR,ko;q=0.9,en-US;q=0.8
```

HTTP 버전: **HTTP/1.1**

## 결론

현대 브라우저(Chrome, Firefox, Safari)와 curl은 모두 **HTTP/1.1**을 사용한다.
Tiny 서버는 HTTP/1.0 서버이지만, 클라이언트의 HTTP 버전과 관계없이 요청을 처리한다.
Tiny는 응답 시 항상 `HTTP/1.0 200 OK`를 반환한다.

요청 라인의 세 번째 토큰(version)이 HTTP 버전이다:

```
GET /home.html HTTP/1.1
 ^       ^        ^
method  URI    version
```

## 관련 코드

doit() 함수에서 요청 라인을 파싱하는 부분:

```c
sscanf(buf, "%s %s %s", method, uri, version);
```

여기서 `version` 변수에 "HTTP/1.1" 또는 "HTTP/1.0"이 저장된다.
현재 Tiny는 이 값을 사용하지 않고 `(void)version;`으로 무시한다.
